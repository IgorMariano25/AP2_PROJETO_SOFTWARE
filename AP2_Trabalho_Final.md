# Trabalho Final — Machine Learning Aplicado à Mineração de Repositórios de Código-Fonte

## Modalidade

Trabalho em dupla.

## Prazo de Entrega

**16/06**, com apresentação.

---

## Objetivo Geral

O objetivo deste trabalho é aplicar conceitos de **Engenharia de Software**, **Ciência de Dados** e **Machine Learning** na análise de repositórios reais de código-fonte, utilizando técnicas de mineração de software para extração de métricas e construção de modelos inteligentes capazes de identificar padrões, realizar classificações ou gerar predições relacionadas à evolução e qualidade de sistemas de software.

---

## Descrição Geral

O trabalho será dividido em três grandes etapas:

1. Extração de dados e métricas de repositórios de código-fonte;
2. Aplicação de algoritmos de Machine Learning;
3. Escrita de artigo científico.

Cada dupla deverá selecionar ao menos **10 repositórios de software reais**, extrair informações relevantes de Engenharia de Software, construir um dataset próprio e desenvolver uma solução baseada em Machine Learning para resolver um problema relacionado à análise de software.

---

## Tema do Trabalho

O tema é livre, desde que esteja relacionado à análise inteligente de repositórios de código-fonte.

### Exemplos de temas possíveis

- Predição de defeitos em arquivos-fonte;
- Classificação de commits de risco;
- Detecção de vulnerabilidades;
- Predição de bugs;
- Predição de falhas em pipelines CI/CD;
- Identificação automática de débito técnico;
- Classificação de severidade de issues;
- Análise de sentimento em comentários de desenvolvedores;
- Predição de rejeição de pull requests;
- Identificação de arquivos propensos a mudanças;
- Detecção de code smells;
- Clusterização de projetos de software;
- Identificação de desenvolvedores especialistas;
- Predição de esforço de manutenção;
- Análise da evolução arquitetural;
- Detecção de anomalias em commits.

---

# Parte 1 — Extração de Dados e Métricas

## Objetivo

A dupla deverá realizar a mineração automatizada de repositórios de código-fonte, extraindo métricas e atributos relevantes para análise posterior.

---

## Fontes Permitidas

Os dados podem ser obtidos a partir de plataformas como:

- GitHub;
- GitLab;
- Bitbucket;
- SonarQube;
- APIs REST;
- Git log;
- Arquivos-fonte;
- Datasets públicos complementares.

---

## Requisitos Obrigatórios

A dupla deverá:

- Utilizar pelo menos **10 repositórios reais**;
- Automatizar a coleta de dados;
- Construir um dataset próprio;
- Documentar o processo de extração.

---

## Exemplos de Métricas

Exemplos de atributos que podem ser utilizados:

- Número de commits;
- Linhas de código;
- Churn de código;
- Complexidade ciclomática;
- Número de autores;
- Frequência de modificações;
- Quantidade de bugs;
- Cobertura de testes;
- Dependências;
- Comentários;
- Métricas CK;
- Métricas de segurança;
- Tempo entre commits;
- Número de pull requests;
- Quantidade de issues;
- Tamanho de arquivos;
- Histórico de manutenção.

---

# Parte 2 — Aplicação de Machine Learning

## Objetivo

A dupla deverá utilizar os dados extraídos para desenvolver modelos de Machine Learning capazes de resolver o problema escolhido.

---

## Requisitos Obrigatórios

O projeto deverá incluir:

- Tratamento e limpeza de dados;
- Engenharia de atributos;
- Divisão treino/teste;
- Treinamento de modelos;
- Comparação entre algoritmos;
- Avaliação experimental.

---

## Algoritmos

A dupla deverá aplicar pelo menos um algoritmo de Machine Learning, tais como:

- Regressão Linear;
- Regressão Logística;
- Árvores de Decisão;
- Random Forest;
- XGBoost;
- SVM;
- Redes Neurais;
- KNN;
- Naive Bayes;
- Gradient Boosting;
- LightGBM.

---

## Avaliação

Os modelos deverão ser avaliados utilizando métricas apropriadas, tais como:

- Accuracy;
- Precision;
- Recall;
- F1-Score;
- ROC-AUC;
- RMSE;
- MAE;
- Matriz de Confusão.

---
